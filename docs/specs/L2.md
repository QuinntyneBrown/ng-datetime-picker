# L2 Requirements — DateTime Picker

## L2-1: Component Structure (traces to L1-1)

The datetime picker component shall be a standalone Angular component with selector `qb-datetime-picker`. It shall replace the existing placeholder `NgDatetimePicker` component.

**Acceptance Criteria:**
- AC-1: Component has selector `qb-datetime-picker`
- AC-2: Component is standalone
- AC-3: Component is exported from the library public API

## L2-2: Date Selection (traces to L1-1)

The component shall include an Angular Material datepicker (`mat-datepicker`) that allows the user to select a date.

**Acceptance Criteria:**
- AC-1: A `mat-datepicker` is rendered in the template
- AC-2: Selecting a date updates the component's value

## L2-3: Time Selection (traces to L1-1)

The component shall include time inputs (hour and minute) that allow the user to select a time.

**Acceptance Criteria:**
- AC-1: Hour and minute input fields are rendered
- AC-2: Changing hour/minute updates the component's value

## L2-4: Combined DateTime Value (traces to L1-1)

The component shall combine the selected date and time into a single `Date` output. It shall support two-way binding via an `EventEmitter`.

**Acceptance Criteria:**
- AC-1: Component has a `value` input of type `Date | null`
- AC-2: Component emits a `valueChange` event when the datetime changes
- AC-3: When `value` is set from outside, date and time fields reflect it

## L2-5: Angular Material Dependencies (traces to L1-1)

The workspace shall include `@angular/material` and `@angular/cdk` as dependencies. The library `package.json` shall declare them as peer dependencies.

**Acceptance Criteria:**
- AC-1: `@angular/material` and `@angular/cdk` are installed
- AC-2: Library `package.json` lists them as peer dependencies
- AC-3: `@angular/animations` and `@angular/platform-browser/animations` are available

## L2-6: Jest Configuration (traces to L1-2)

The workspace shall have Jest configured with `jest-preset-angular` to compile and run Angular component tests.

**Acceptance Criteria:**
- AC-1: `jest`, `jest-preset-angular`, and `@types/jest` are installed as dev dependencies
- AC-2: A `jest.config.ts` exists at workspace root with proper Angular preset configuration
- AC-3: A `setup-jest.ts` file initializes the Angular testing environment
- AC-4: `npm test` runs Jest and reports results

## L2-7: DateTime Picker Tests — 100% Line Coverage (traces to L1-3)

Unit tests shall cover all lines of the datetime picker component.

**Acceptance Criteria:**
- AC-1: Tests verify component creation
- AC-2: Tests verify date selection updates the value
- AC-3: Tests verify time (hour/minute) changes update the value
- AC-4: Tests verify external `value` input propagates to date and time fields
- AC-5: Tests verify `valueChange` emits the combined datetime
- AC-6: Tests verify null/edge-case handling
- AC-7: Jest coverage report shows 100% line coverage for the component file
