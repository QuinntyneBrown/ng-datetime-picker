# L2 Requirements — DateTime Picker

## L2-1: Component Structure (traces to L1-1)

The datetime picker component shall be a standalone Angular component with selector `qb-datetime-picker`. It shall replace the existing placeholder `NgDatetimePicker` component.

**Acceptance Criteria:**
- AC-1: Component has selector `qb-datetime-picker`
- AC-2: Component is standalone
- AC-3: Component is exported from the library public API

## L2-2: Date Selection (traces to L1-1)

The component shall include an Angular Material datepicker (`mat-datepicker`) that allows the user to select a date.

**Acceptance Criteria:**
- AC-1: A `mat-datepicker` is rendered in the template
- AC-2: Selecting a date updates the component's value

## L2-3: Time Selection (traces to L1-1)

The component shall include time inputs (hour, minute, and second) that allow the user to select a time down to the second.

**Acceptance Criteria:**
- AC-1: Hour, minute, and second input fields are rendered
- AC-2: Changing hour/minute/second updates the component's value

## L2-4: Combined DateTime Value (traces to L1-1)

The component shall combine the selected date and time into a single `Date` output. It shall support two-way binding via an `EventEmitter`.

**Acceptance Criteria:**
- AC-1: Component has a `value` input of type `Date | null`
- AC-2: Component emits a `valueChange` event when the datetime changes
- AC-3: When `value` is set from outside, date and time fields reflect it

## L2-5: Angular Material Dependencies (traces to L1-1)

The workspace shall include `@angular/material` and `@angular/cdk` as dependencies. The library `package.json` shall declare them as peer dependencies.

**Acceptance Criteria:**
- AC-1: `@angular/material` and `@angular/cdk` are installed
- AC-2: Library `package.json` lists them as peer dependencies
- AC-3: `@angular/animations` and `@angular/platform-browser/animations` are available

## L2-6: Jest Configuration (traces to L1-2)

The workspace shall have Jest configured with `jest-preset-angular` to compile and run Angular component tests.

**Acceptance Criteria:**
- AC-1: `jest`, `jest-preset-angular`, and `@types/jest` are installed as dev dependencies
- AC-2: A `jest.config.ts` exists at workspace root with proper Angular preset configuration
- AC-3: A `setup-jest.ts` file initializes the Angular testing environment
- AC-4: `npm test` runs Jest and reports results

## L2-7: DateTime Picker Tests — 100% Line Coverage (traces to L1-3)

Unit tests shall cover all lines of the datetime picker component.

**Acceptance Criteria:**
- AC-1: Tests verify component creation
- AC-2: Tests verify date selection updates the value
- AC-3: Tests verify time (hour/minute) changes update the value
- AC-4: Tests verify external `value` input propagates to date and time fields
- AC-5: Tests verify `valueChange` emits the combined datetime
- AC-6: Tests verify null/edge-case handling
- AC-7: Jest coverage report shows 100% line coverage for the component file

## L2-8: Second Input Field (traces to L1-4)

The component shall include a second input field (0-59) that allows the user to select seconds.

**Acceptance Criteria:**
- AC-1: A "Second" number input field is rendered in the template with min 0 and max 59
- AC-2: Changing the second value calls `onSecondChange` and updates the `second` property
- AC-3: The second value is clamped to the range 0-59
- AC-4: The combined datetime output includes the selected seconds
- AC-5: When `value` is set from outside, the second field reflects `value.getSeconds()`
- AC-6: When value is null, second resets to 0
- AC-7: Tests cover second selection, clamping, null handling, and template rendering with 100% line coverage

## L2-10: UTC ISO String Output (traces to L1-6)

The component's `value` input and `valueChange` output shall use `string | null` type, where the string is a UTC ISO 8601 datetime (e.g., `"2024-01-15T14:30:00.000Z"`).

**Acceptance Criteria:**
- AC-1: The `value` input accepts `string | null` (UTC ISO format)
- AC-2: The `valueChange` output emits `string | null` in UTC ISO format (ending in `Z`)
- AC-3: When a date and time are selected, the emitted string represents the selected datetime converted to UTC
- AC-4: When `value` is set from outside with an ISO string, date and time fields reflect the correct local values
- AC-5: When `value` is null, the output is null
- AC-6: The playground app displays the UTC ISO string output
- AC-7: Playwright e2e tests verify the output is a valid UTC ISO 8601 datetime string

## L2-9: Outline Appearance for Form Fields (traces to L1-5)

All `mat-form-field` elements in the datetime picker template shall use `appearance="outline"`.

**Acceptance Criteria:**
- AC-1: Every `<mat-form-field>` in the template has `appearance="outline"`
- AC-2: A test verifies all rendered `mat-form-field` elements have the outline appearance CSS class (`mat-mdc-form-field--outlined` or `mat-form-field-appearance-outline`)

## L2-11: Millisecond Input Field (traces to L1-7)

The component shall include a millisecond input field (0-999) that allows the user to select milliseconds.

**Acceptance Criteria:**
- AC-1: A "Millisecond" number input field is rendered in the template with min 0 and max 999
- AC-2: Changing the millisecond value calls `onMillisecondChange` and updates the `millisecond` property
- AC-3: The millisecond value is clamped to the range 0-999
- AC-4: The combined datetime output includes the selected milliseconds
- AC-5: When `value` is set from outside, the millisecond field reflects `value.getMilliseconds()`
- AC-6: When value is null, millisecond resets to 0
- AC-7: Tests cover millisecond selection, clamping, null handling, and template rendering with 100% line coverage
- AC-8: The total number of `mat-form-field` elements is now 5 (date, hour, minute, second, millisecond)
- AC-9: The total number of `input[type="number"]` elements is now 4 (hour, minute, second, millisecond)

## L2-12: DateTime Picker Design Layout (traces to L1-8)

The `.pen` design shall show the `qb-datetime-picker` component with a horizontal row of five outlined form fields: Date (with calendar icon), Hour, Minute, Second, and Millisecond.

**Acceptance Criteria:**
- AC-1: A container frame holds all five form fields in a horizontal row layout
- AC-2: Each form field has a visible outline border (Angular Material outline appearance style)
- AC-3: Each field has a floating label: "Date", "Hour", "Minute", "Second", "Millisecond"
- AC-4: The Date field includes a calendar icon toggle on the right side
- AC-5: The Hour, Minute, Second, and Millisecond fields show sample numeric values
- AC-6: Fields are evenly spaced with consistent gaps between them
- AC-7: The overall design uses a clean, modern aesthetic consistent with Angular Material theming

## L2-13: DateTime Picker Design Title and Output Display (traces to L1-8)

The design shall include a title above the picker and an output display below it showing a sample UTC ISO datetime string.

**Acceptance Criteria:**
- AC-1: A "DateTime Picker" title is displayed above the form fields
- AC-2: Below the picker, a "Selected:" label shows a sample ISO datetime string (e.g., "2024-01-15T14:30:45.123Z")
- AC-3: The title and output text are clearly readable and properly aligned with the picker
- AC-4: The layout has no overlapping elements, clipped content, or misalignment

## L2-14: Compact Input Fields (traces to L1-9)

The compact variant shall use smaller input fields with reduced padding (4-6px vertical, 8px horizontal), smaller font sizes (12px values, 10px labels), and tighter gaps (8px between fields) compared to the standard variant.

**Acceptance Criteria:**
- AC-1: Each input field has reduced internal padding (approximately 4-6px vertical, 8px horizontal)
- AC-2: Value font size is 12-13px (down from 16px) and label font size is 10px (down from 12px)
- AC-3: Gap between fields in the row is 8px or less (down from 12px)
- AC-4: Overall container padding is reduced (16-24px, down from 40-48px)
- AC-5: The total height of the picker row is noticeably shorter than the standard variant
- AC-6: All labels and values remain legible at the reduced sizes

## L2-15: Compact Variant Layout and Title (traces to L1-9)

The compact variant shall be displayed as a separate screen in the `.pen` file, labeled "DateTime Picker — Compact", placed next to the existing standard variant. It shall include the same output display but with proportionally reduced text sizes.

**Acceptance Criteria:**
- AC-1: A separate frame labeled "DateTime Picker — Compact" exists in the document
- AC-2: The compact frame is placed to the right of or below the existing standard variant without overlap
- AC-3: Title font size is reduced (18-20px, down from 28px)
- AC-4: Output display text is reduced (12px, down from 14px)
- AC-5: The compact variant total width is narrower than the standard variant (target ~560-620px)
- AC-6: No overlapping elements, clipped content, or misalignment

## L2-16: Compact Input Property (traces to L1-10)

The component shall accept a boolean `compact` input (default `false`). When `true`, the CSS class `qb-datetime-picker--compact` is added to the container div.

**Acceptance Criteria:**
- AC-1: The component has a `@Input() compact: boolean = false` property
- AC-2: When `compact` is `false`, the container div has only the class `qb-datetime-picker`
- AC-3: When `compact` is `true`, the container div has both `qb-datetime-picker` and `qb-datetime-picker--compact` classes
- AC-4: Tests verify the compact class is toggled based on the input

## L2-17: Compact CSS Styling (traces to L1-10)

The `.qb-datetime-picker--compact` class shall reduce the gap between fields and apply Angular Material density reduction to make form fields smaller.

**Acceptance Criteria:**
- AC-1: The `.qb-datetime-picker--compact` class reduces the gap to 4px (from 8px)
- AC-2: The `.qb-datetime-picker--compact` class applies `--mat-form-field-container-vertical-padding: 4px` and `--mat-form-field-container-height: 40px` to shrink form fields
- AC-3: The compact variant is visually shorter and narrower than the standard variant
- AC-4: Tests verify compact mode renders with the correct CSS class

## L2-18: Compact Mode Unit Tests (traces to L1-10)

Unit tests shall cover the compact mode functionality with 100% line coverage maintained.

**Acceptance Criteria:**
- AC-1: Tests verify `compact` defaults to `false`
- AC-2: Tests verify setting `compact = true` adds the `qb-datetime-picker--compact` class to the DOM
- AC-3: Tests verify setting `compact = false` does not add the compact class
- AC-4: Jest coverage report maintains 100% line coverage
