# L1 Requirements — DateTime Picker

## L1-1: DateTime Picker Component

The library shall provide a standalone Angular component that allows users to select both a date and a time, down to the second. The component shall use Angular Material UI components with outline appearance and expose the selector `qb-datetime-picker`.

## L1-5: Outline Form Field Appearance

All `mat-form-field` elements in the datetime picker shall use `appearance="outline"`.

## L1-4: Second-Precision Time Selection

The datetime picker shall support time selection with second-level granularity in addition to hours and minutes.

## L1-2: Jest Test Infrastructure

The library shall use Jest as its unit test runner with configuration compatible with Angular 17.2. All component code shall have 100% line coverage.

## L1-3: DateTime Picker Unit Tests

The library shall include unit tests that exercise every line of the datetime picker component, achieving 100% line coverage.

## L1-6: UTC ISO DateTime Output Format

The datetime picker component shall emit its output value as a UTC ISO 8601 datetime string (e.g., `"2024-01-15T14:30:00.000Z"`) rather than a `Date` object. The `value` input and `valueChange` output shall use `string | null`.

## L1-7: Millisecond-Precision Time Selection

The datetime picker shall support time selection with millisecond-level granularity (0-999) in addition to hours, minutes, and seconds.

## L1-8: Visual Design of DateTime Picker

A `.pen` design file shall visually represent the `qb-datetime-picker` component, showing its layout with all fields (date, hour, minute, second, millisecond) using Angular Material outline-style form fields. The design shall serve as a reference for the component's visual appearance.

## L1-9: Compact Dashboard Variant Design

The `.pen` design file shall include a compact variant of the datetime picker optimized for dense dashboard layouts with many fields and tables. Input boxes shall use reduced padding, smaller font sizes, and tighter spacing to minimize vertical and horizontal footprint.

## L1-10: Compact Mode Implementation

The `qb-datetime-picker` component shall support a `compact` input that, when set to `true`, applies the CSS class `qb-datetime-picker--compact` to the container. This class reduces form field density via tighter gaps and Angular Material density styling, making the picker suitable for dense dashboard layouts.

## L1-11: Short-Form Time Labels and Reduced Input Width

The datetime picker shall use short-form abbreviations for time field labels (Hr, Min, Sec, Ms) instead of full words (Hour, Minute, Second, Millisecond), and the time input fields shall be 50% narrower than their current width.
